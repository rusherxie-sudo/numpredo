# 技巧连动提示强化（第一层 ①）设计

> 目标：把可玩数独的「ヒント」从"直接揭示答案"升级为**三级递进的技巧教学提示**，让玩家在解题中真正学到技巧。这是「深度」轴上超过 sudoku.com 的核心一手——既是游戏功能、又是 SEO 内部链接与滞在时间来源。北极星：Google 自然流量获客（教学深度 = 参与度信号 + 技巧页内链）。

## 背景与约束

- 现状 `hint()`（`src/client/sudoku-game.ts:366`）三步：①查错填格 → ②找単数指路（带技巧名+链接）→ ③单数推不动时**直接揭示答案**配泛链接。第③步最弱，浪费了引擎的高级技巧链。
- 引擎 `traceFirstElimination(grid, ctx)`（已从 `src/engine/index.ts` 导出）沿技巧链 `nakedSingle→hiddenSingle→lockedCandidates→nakedPair→hiddenPair→nakedTriple→xWing`（正好对齐五档难度）返回「第一个消候选步」的执行前盘面 + 候选数组 + `SolveStep`。
- **关键约束**：高级技巧本质是「消候选」，而本站游戏只有**手动笔记、无自动候选显示**（自动候选属第二层，冻结）。所以高级技巧提示必须**临时显示引擎现算的候选浮层**，玩家才看得懂"消哪个候选"。
- `SolveStep` 对消除型只给 `eliminations: [cell,digit][]`（被消候选），**不给"构成技巧的基准格"**。故 v1 高亮「被消候选的目标格 + 所在单元」，不追求精确圈出 X-Wing 四角——教学上够用且一定拿得到。
- 全部改动在客户端：**零引擎改动、零题库再生、零内容页改动**。

## 交互设计：三级递进状态机

用 `hintLevel`（1–3）+ `hintSig`（当前 `cur` 盘面串）驱动：
- 每次点提示，先算盘面签名 `sig = cur.join('')`。`sig !== hintSig` → 说明玩家动过盘（落子/撤销/清格/换题/恢复）→ 重置 `hintLevel=1`、`hintSig=sig`；否则（连点未动盘）→ `hintLevel = min(hintLevel+1, 3)`。
- **纠错优先且不占级别**：若存在错填格，无论级别都只高亮报「◯マス間違っています」并返回（玩家改对后盘面变、签名自动重置）。

确定「下一步」：先 `findSingle()`（复用现有，找 naked/hidden single）；找不到再 `traceFirstElimination` 拿高级技巧步。按级别展开该步：

| 级别 | 単数（single） | 消除型（elimination） |
|---|---|---|
| **1 ナッジ** | 高亮目标格所在宫（`sk-hint-region`），文案「光っているブロックに次の一手があります。もう一度ヒントで詳しく。」不报技巧名、不给答案 | 高亮消除目标格所在宫，同上文案 |
| **2 テクニック** | 高亮目标格 + 候选浮层（该格候选）+ 技巧名 + `解き方`链接，「<b>◯◯</b>が使えます。ハイライトした候補に注目（解き方）。もう一度ヒントで答え。」不直说数字 | 高亮消除目标格 + 候选浮层（引擎返回的 `candidates` 铺在这些格）+ 技巧名 + 链接 |
| **3 結論** | 「このマスは<b>◯◯</b>で <b>N</b> に決まります（解き方）。」揭示数字 | 「<b>◯◯</b>で、赤い候補（<b>数字</b>）を消せます（解き方）。」候选浮层里被消数字标红（`sk-hx`） |

## 候选浮层（新增渲染）

- 新增 hint 视觉状态（module 内）：`hintRegion:Set<number>`（区域）、`hintFocus:Set<number>`（目标格）、`hintWrong:Set<number>`（错填格）、`hintCand:Map<number,number>`（格→要显示的候选 bitmask）、`hintX:Map<number,number>`（格→要标红/删除的候选 bitmask）。
- `clearHint()` 清空以上全部 + 消息文本。任何落子/撤销/清格/方向键/点格/换题都调 `clearHint()`（已有调用点）。
- `render()` 集成：区域格加 `sk-hint-region`、目标格加 `sk-hint-cell`、错填格加 `sk-wrong-cell`；单元格内容分支新增——空格且 `hintCand.has(i)` 时，渲染 `sk-notes sk-hint-notes` 浮层（与玩家笔记视觉区分），其中 `hintX` 命中的数字加 `sk-hx`（红/删除线）。绝不写进玩家 `notes`。
- 浮层候选来源：单数走 `findSingle` 内已算的 `cand`；消除型直接用 `traceFirstElimination` 返回的 `candidates[cell]`——均来自引擎真实推理，且传 `ctx` **自动兼容对角线变体**。
- `render()` 里 className 的 `sk-area-done` 保留（regex-keep 收窄到只保 area-done，其余 hint 类由上述 Set 驱动，选中/移动重渲不再丢失高亮）。

## 兜底 / 边界

- 错填格存在 → 纠错分支，不升级、不算下一步。
- 已解 / 无空格 → 空操作（保留现有 `done` guard）。
- `findSingle` 与 `traceFirstElimination` 都返回 null（本站题保证纯逻辑可解，理论不发生）→ 回退到现状③：揭示第一个空格的解答数字作安全网。
- 对角线变体 → 传 `DIAGONAL` ctx（`setup` 里已有 `ctx`），单元/候选全部对角线制约込み。
